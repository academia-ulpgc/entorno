# Guion temporal V.4 - Automatizacion sencilla y entrega del trabajo

## Uso

Documento para el docente. No entregar como ficha de estudiante.

## Duracion

1 hora y 15 minutos.

## Objetivo de conduccion

Cerrar el bloque mostrando una automatizacion sencilla: datos ficticios de entrada, script Python como instrucciones, informe en texto plano como salida y carpeta de entrega revisada.

## Guion temporal

- 0:00-0:10. Repaso:
  - Recuperar ideas: entrada, instrucciones, salida, revision.
  - Explicar que automatizar no significa perder control.

- 0:10-0:30. Datos y script:
  - Crear `solicitudes.csv`.
  - Crear `generar_informe.py`.
  - Explicar brevemente que el CSV son datos y el script son instrucciones.

- 0:30-0:45. Ejecucion:
  - Ejecutar el script.
  - Abrir `informe_solicitudes.txt`.
  - Comparar con LaTeX: aqui no compilamos documento, ejecutamos instrucciones Python.

- 0:45-0:58. Validacion:
  - Revisar totales, estados y areas.
  - Completar `notas-validacion.txt`.
  - Reforzar uso de datos ficticios.

- 0:58-1:10. Entrega:
  - Crear `entrega-bloque-v`.
  - Completar `ENTREGA.txt`.
  - Revisar que no haya datos reales, contraseñas ni informacion sensible.

- 1:10-1:15. SSO e incidencias:
  - Documentar incidencias de acceso, SSO o Copilot.
  - Cerrar con la idea: la IA ha actuado como apoyo tecnico, pero la persona ha dirigido y validado.

## Mensaje clave

Una automatizacion sencilla tiene tres piezas visibles: datos de entrada, instrucciones y resultado. La calidad depende de revisar cada una.
